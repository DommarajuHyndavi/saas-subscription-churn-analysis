# saas-subscription-churn-analysis
SQL &amp; Power BI project analyzing SaaS subscription data to track MRR, ARR, ARPU, and customer churn.


## Project Overview
This project analyzes a SaaS subscription dataset to evaluate customer retention and business performance using SQL and Power BI. The objective is to identify churn trends, monitor recurring revenue, and generate actionable insights through interactive dashboards.


## Business Problem

A SaaS company wants to understand customer retention, subscription performance, and recurring revenue by analyzing subscription data and identifying churn trends.


## Objectives
- Calculate Monthly Recurring Revenue (MRR)
- Calculate Annual Recurring Revenue (ARR)
- Measure Customer Churn Rate
- Analyze Average Revenue Per User (ARPU)
- Monitor Subscription Growth
- Track Upgrade and Downgrade Trends
- Evaluate Auto-Renew Performance

## Dataset
The analysis uses a SaaS Subscription Analytics dataset consisting of:
- Accounts
- Subscriptions
- Churn Events
- Feature Usage
- Support Tickets


## Dashboard KPIs
- Total Customers
- Active Customers
- Churned Customers
- Monthly Recurring Revenue (MRR)
- Annual Recurring Revenue (ARR)
- Average Revenue Per User (ARPU)
- Churn Rate
- Auto Renew Subscriptions
- Upgrade Count
- Downgrade Count


## Key Insights

- Enterprise plans generated the highest recurring revenue.
- Customers with Auto-Renew had significantly lower churn.
- Premium plans produced the highest ARPU.
- Most churn occurred among monthly subscription customers.
- Upgraded customers contributed more to overall MRR growth.
